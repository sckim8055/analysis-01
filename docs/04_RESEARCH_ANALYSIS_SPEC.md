# 연구 분석 기능 명세

> 본 문서는 SPSS 26 및 PROCESS Macro v5를 Python으로 대체하는 통계 분석 기능의 상세 명세를 정의한다.  
> 분석 파이프라인, 각 분석의 입출력, 알고리즘 기준을 포함한다.

---

## 1. 분석 파이프라인 개요

```
[1단계] 데이터 클린징         ← 필수, 최초 1회
    ↓
[2단계] 변수-요인 매핑        ← 필수, 클린징 후
    ↓
[3단계] 요인분석 → 확정       ← 필수, 매핑 후
    ↓
[4단계] 분석 수행 (동일 레벨)  ← 탭별 독립 실행, 순서 무관
    ├─ 빈도분석
    ├─ 신뢰도 분석
    ├─ 상관관계 분석
    ├─ 독립표본 T검정
    ├─ ANOVA 분석
    ├─ 다중회귀분석
    ├─ 매개효과 분석
    └─ 조절효과 분석
    ↓
[5단계] AI 해석 + 보고서 출력  ← 선택적
```

> **핵심 규칙:** 1~3단계는 순차적 의존성이 있으나, 4단계의 8개 분석은 선후 관계 없이 독립 실행 가능하다.

## 2. 1단계: 데이터 클린징

### 2.1 클린징 작업 목록

| 작업 | 설명 | 사용자 설정 |
|---|---|---|
| 결측치 탐지 | 빈 셀, NA, 무응답 탐지 | 처리 방법 선택 (삭제/평균대체/중앙값대체) |
| 이상치 탐지 | Z-score 기반 이상치 식별 | 기준값 설정 (기본: ±3.0) |
| 역코딩 | 부정 문항 역코딩 처리 | 역코딩 대상 항목 지정, 척도 범위 입력 |
| 데이터 타입 검증 | 숫자형/문자형 확인 | 자동 |
| 중복 응답 탐지 | 동일 패턴 반복 응답 식별 | 제거 여부 선택 |
| 불성실 응답 탐지 | 표준편차 0인 응답 등 | 기준 설정 |

### 2.2 입력/출력

```
입력:
  - 원본 데이터 파일 (Excel/CSV)
  - 클린징 규칙 설정 (사용자 입력)

출력:
  - 정제된 데이터셋
  - 클린징 보고서 (제거/대체된 항목 목록, 전후 비교)
```

### 3.1 데이터 소스

업로드 대상은 **Google Sheets**, **네이버 설문지** 등에서 수집한 설문 데이터 파일이다.

### 3.2 변수 계층 구조 (4단계)

> **핵심:** 모든 변수 유형은 **여러 개** 일 수 있고, 각 변수의 **하위변수도 여러 개** 일 수 있으며,
> 요인분석 후 하위변수는 **여러 요인**으로 그룹핑된다.

```
계층 구조:

[Level 1] 변수 유형 (Variable Type)
    ├─ 독립변수 (IV)     ← 여러 개 가능
    ├─ 종속변수 (DV)     ← 여러 개 가능
    ├─ 매개변수 (MED)    ← 여러 개 가능
    └─ 조절변수 (MOD)    ← 여러 개 가능

[Level 2] 변수 (Variable)          ← 각 유형당 여러 개
    예: 서비스 품질, 가격 적정성, ...

[Level 3] 하위변수 (Sub-variable / Item)   ← 각 변수당 여러 개
    예: SQ1, SQ2, SQ3, SQ4 (= 엑셀 컬럼)

[Level 4] 요인 (Factor) — 요인분석 후 생성
    예: 요인1(서비스 응대): SQ1,SQ2 / 요인2(서비스 환경): SQ3,SQ4
```

#### 조작 방법

| 조작 | 방법 |
|---|---|
| **컬럼 → 변수 영역으로 드래그** | 엑셀 컬럼을 원하는 변수 유형 영역(IV/DV/MED/MOD)에 드래그하여 배치 |
| **새 변수 그룹 생성** | 변수 유형 영역 내에서 "📁 + 새 변수 추가" 버튼 클릭 → 변수명 입력 |
| **변수 그룹에 컬럼 배치** | 드래그한 컬럼을 특정 변수 그룹(📂) 안에 드롭 |
| **다중 선택 드래그** | Ctrl+클릭으로 여러 컬럼 선택 후 한번에 드래그 |
| **변수 그룹 간 이동** | 이미 배치된 컬럼을 다른 변수 그룹으로 드래그하여 이동 |
| **배치 해제** | 배치된 컬럼을 왼쪽 컬럼 목록으로 다시 드래그하거나 ✕ 버튼 |
| **변수 그룹명 수정** | 변수 그룹명 더블클릭 → 인라인 편집 |
| **변수 그룹 삭제** | 변수 그룹 우클릭 → 삭제 (하위 컬럼은 미배정으로 복귀) |

### 3.5 매핑 규칙

| 규칙 | 설명 |
|---|---|
| 한 컬럼은 하나의 변수 그룹에만 속함 | 중복 매핑 불가 |
| 변수 그룹당 최소 1개 이상의 하위변수 필요 | 빈 그룹 불가 |
| 변수 유형당 변수 그룹 수 제한 없음 | 독립변수가 5개여도 OK |
| 매핑 완료 후 수정 가능 | 요인분석 전까지 자유 수정 |
| 매핑 상태 자동 저장 | 드래그 즉시 DB 반영, 새로고침 시 유지 |
| ID, 인구통계 컬럼 | 분석 대상이 아닌 컬럼은 "제외" 영역에 배치 가능 |

### 3.6 요인분석 후 구조 변화

```
[매핑 단계] 2레벨 구조:
  변수 그룹 → 하위변수(컬럼)
  서비스 품질 → [SQ1, SQ2, SQ3, SQ4, SQ5, SQ6]

     ↓ 요인분석 실행 ↓

[요인분석 후] 3레벨 구조:
  변수 그룹 → 요인 → 하위변수(컬럼)
  서비스 품질 → 요인1 "서비스 응대" → [SQ1, SQ2, SQ4]
             → 요인2 "서비스 환경" → [SQ5, SQ6]
             (SQ3 제거됨)
```

> **중요:** 4단계 분석(신뢰도, 상관 등)에서는 **요인 단위**로 분석이 진행된다.
> 즉, "서비스 품질" 전체가 아니라 "서비스 응대(요인1)"와 "서비스 환경(요인2)"이
> 각각 독립적인 분석 단위가 된다.

## 4. 3단계: 요인분석

### 4.1 분석 옵션 (사용자 설정 가능)

| 옵션 | 설명 | 기본값 | 범위 |
|---|---|---|---|
| 요인추출 방법 | EFA(탐색적 성분) / PCA(주성분) | PCA | 선택 |
| 요인회전 방법 | Varimax (직교회전) | Varimax | 선택 |
| 고유값 기준 | Eigenvalue 최소값 | 1.0 | 0.1 ~ 5.0 |
| 요인적재값 기준 | Factor Loading 최소값 | 0.5 | 0.1 ~ 0.9 |
| 공통성 기준 | Communality 최소값 | 0.4 | 0.1 ~ 0.9 |
| 분산설명력 기준 | Explained Variance 최소값 | 60% | 10% ~ 100% |
| KMO 기준 | KMO 최소값 | 0.5 | 0.0 ~ 1.0 |

### 4.2 자동 정리 알고리즘 (Auto-Refine)

> **핵심 원칙:** 사용자가 설정한 기준치(고유값, 요인적재값, 공통성, 분산설명력, KMO)를
> **모두 만족할 때까지** 시스템이 자동으로 항목 제거 및 요인 수 조정을 반복한다.
> 사용자는 최종 결과만 확인하고 확정/수동조정을 선택한다.

#### 전체 흐름

```
사용자가 기준치 설정 → [자동 분석 시작]
                            │
                    ┌───────▼────────┐
                    │ STEP 1         │
                    │ 초기 요인분석   │ ← 전체 항목으로 첫 실행
                    │ 실행           │    요인 수는 고유값 기준으로 자동 결정
                    └───────┬────────┘
                            │
                    ┌───────▼────────┐
                    │ STEP 2         │
                    │ 기준치 체크    │ ← 7개 기준 전부 확인
                    │                │
                    └───┬────┬───────┘
                   통과 │    │ 미달
                        │    │
             ┌──────────▼┐ ┌─▼──────────────┐
             │ 완료!     │ │ STEP 3          │
             │ 결과 제시 │ │ 위반 항목 식별  │
             └───────────┘ │ 제거 대상 결정  │
                           └───────┬─────────┘
                                   │
                           ┌───────▼─────────┐
                           │ STEP 4           │
                           │ 항목 1개 제거    │ ← 가장 심각한 위반 항목
                           │ (또는 요인 조정) │
                           └───────┬─────────┘
                                   │
                                   └──→ STEP 1로 돌아가 재실행
```

#### 상세 의사코드

```python
def auto_refine_factor_analysis(data, items, options):
    """
    사용자 기준치를 모두 만족할 때까지 자동 반복
    
    options:
      extraction_method: 'pca' | 'efa'
      rotation_method: 'varimax'
      eigenvalue_threshold: 1.0      # 고유값 기준
      loading_threshold: 0.5         # 요인적재값 기준
      communality_threshold: 0.4     # 공통성 기준
      variance_threshold: 60.0       # 분산설명력 기준 (%)
      kmo_threshold: 0.5             # KMO 기준
    """
    
    current_items = items.copy()
    iteration = 0
    max_iterations = 50  # 무한루프 방지
    history = []  # 매 반복의 결과 기록 (되돌리기용)
    
    while iteration < max_iterations:
        iteration += 1
        
        # ═══════════════════════════════════════════
        # STEP 1: 현재 항목으로 요인분석 실행
        # ═══════════════════════════════════════════
        # 요인 수는 고유값 기준(eigenvalue >= threshold)으로 자동 결정
        n_factors = count_factors_by_eigenvalue(data[current_items], options)
        
        result = run_factor_analysis(
            data=data[current_items],
            n_factors=n_factors,
            method=options['extraction_method'],
            rotation=options['rotation_method']
        )
        
        # 반복 이력 저장 (WebSocket으로 실시간 전송)
        history.append({
            'iteration': iteration,
            'items': current_items.copy(),
            'n_factors': n_factors,
            'kmo': result.kmo,
            'variance': result.total_variance,
            'removed': None
        })
        
        # ═══════════════════════════════════════════
        # STEP 2: 모든 기준치 체크
        # ═══════════════════════════════════════════
        violations = []
        
        # (a) KMO 체크
        if result.kmo < options['kmo_threshold']:
            violations.append({
                'type': 'kmo_low',
                'value': result.kmo,
                'threshold': options['kmo_threshold']
            })
        
        # (b) 공통성 미달 항목 찾기
        for item, comm in result.communalities.items():
            if comm < options['communality_threshold']:
                violations.append({
                    'type': 'communality_low',
                    'item': item,
                    'value': comm,
                    'threshold': options['communality_threshold']
                })
        
        # (c) 요인적재값 미달 항목 찾기 (어떤 요인에도 기준 이상 적재 안 됨)
        for item in current_items:
            max_loading = max(abs(result.loadings[item]))
            if max_loading < options['loading_threshold']:
                violations.append({
                    'type': 'loading_low',
                    'item': item,
                    'value': max_loading,
                    'threshold': options['loading_threshold']
                })
        
        # (d) 교차적재 항목 (2개 이상 요인에 기준 이상 적재)
        for item in current_items:
            high_loadings = [
                abs(l) for l in result.loadings[item]
                if abs(l) >= options['loading_threshold']
            ]
            if len(high_loadings) >= 2:
                violations.append({
                    'type': 'cross_loading',
                    'item': item,
                    'loadings': high_loadings
                })
        
        # (e) 분산설명력 체크
        variance_ok = result.total_variance >= options['variance_threshold']
        
        # ═══════════════════════════════════════════
        # 모든 기준 통과 → 종료
        # ═══════════════════════════════════════════
        item_violations = [v for v in violations if 'item' in v]
        
        if len(item_violations) == 0 and variance_ok:
            # 🎉 모든 기준 충족! 결과 반환
            return AutoRefineResult(
                success=True,
                final_result=result,
                iterations=iteration,
                history=history,
                removed_items=list(set(items) - set(current_items))
            )
        
        # ═══════════════════════════════════════════
        # STEP 3: 제거 대상 결정 (우선순위)
        # ═══════════════════════════════════════════
        remove_target = select_worst_item(item_violations)
        
        # ═══════════════════════════════════════════
        # STEP 4: 항목 제거 후 재실행
        # ═══════════════════════════════════════════
        if remove_target:
            current_items.remove(remove_target['item'])
            history[-1]['removed'] = remove_target
        else:
            # 제거할 항목이 없는데 기준 미달 → 기준 완화 필요
            return AutoRefineResult(
                success=False,
                message="기준치를 만족하는 요인 구조를 찾을 수 없습니다. 기준치 완화를 권장합니다.",
                history=history
            )
        
        # 최소 항목 수 체크 (요인당 최소 2개)
        if len(current_items) < n_factors * 2:
            return AutoRefineResult(
                success=False,
                message=f"항목이 부족합니다. 현재 {len(current_items)}개, 최소 {n_factors * 2}개 필요.",
                history=history
            )
    
    # max_iterations 초과
    return AutoRefineResult(success=False, message="최대 반복 횟수 초과", history=history)
```

#### 항목 제거 우선순위

위반 항목이 여러 개일 때, 아래 우선순위로 **가장 심각한 1개만** 제거한 뒤 재분석한다:

| 우선순위 | 위반 유형 | 제거 기준 |
|:---:|---|---|
| 1 | **공통성 미달** | 공통성이 가장 낮은 항목 1개 제거 |
| 2 | **교차적재** | 2개 이상 요인에 높은 적재값, 차이가 가장 작은 항목 1개 제거 |
| 3 | **요인적재값 미달** | 최대 적재값이 가장 낮은 항목 1개 제거 |

> **왜 1개씩만?** 한번에 여러 개를 제거하면 요인 구조가 급변하여 오히려 결과가 불안정해진다.
> 1개 제거 후 재분석하면 다른 항목의 적재값이 상승하여 기준을 통과할 수 있다.

#### 요인 수 자동 조정

요인 수는 매 반복마다 고유값(Eigenvalue)을 기준으로 **자동 재결정**된다:

```
항목 제거 → 재분석 → 고유값 재계산
  → 고유값 >= 1.0인 요인 수가 변할 수 있음
  → 예: 6개 항목일 때 요인 2개 → 항목 1개 제거 → 5개 항목 → 요인 1개로 줄어들 수 있음
  → 또는: 교차적재 항목 제거 후 → 요인 구조가 깨끗해져 → 요인 수 유지
```

#### 실행 예시

```
[자동 분석 시작] 항목 6개: SQ1, SQ2, SQ3, SQ4, SQ5, SQ6

반복 1: 6개 항목 → 요인 2개 추출
  ├─ KMO = 0.782 ✅ (≥ 0.5)
  ├─ 분산설명력 = 58.2% ❌ (< 60%)
  ├─ 공통성 미달: SQ3 = 0.312 ❌ (< 0.4)
  ├─ 적재값 미달: SQ3 = 0.298 ❌ (< 0.5)
  └─ 제거: SQ3 (공통성 최저)

반복 2: 5개 항목 → 요인 2개 추출
  ├─ KMO = 0.815 ✅
  ├─ 분산설명력 = 63.7% ✅ (≥ 60%)
  ├─ 공통성: 모두 ≥ 0.4 ✅
  ├─ 적재값: SQ5 = 0.472 ❌ (< 0.5)
  └─ 제거: SQ5 (적재값 최저)

반복 3: 4개 항목 → 요인 1개로 조정 (고유값 변화)
  ├─ KMO = 0.842 ✅
  ├─ 분산설명력 = 71.3% ✅
  ├─ 공통성: 모두 ≥ 0.4 ✅
  ├─ 적재값: 모두 ≥ 0.5 ✅
  └─ ✅ 모든 기준 충족! → 결과 제시

[자동 분석 완료]
  최종: SQ1(0.823), SQ2(0.791), SQ4(0.756), SQ6(0.734)
  제거됨: SQ3(공통성 미달), SQ5(적재값 미달)
  요인 수: 2개 → 1개로 자동 조정됨
```

| 분석 방법 | 위계적 회귀 + 상호작용항 |
| 상호작용항 | 독립변수 × 조절변수 (평균중심화 후) |
| 단순 기울기 | 조절변수 수준별 (M-1SD, M, M+1SD) 효과 분석 |
| 자동 판별 | 연구모형 빌더에서 X→Y에 W가 조절하는 구조 감지 시 자동 |
| Python 구현 | `statsmodels OLS` + 상호작용항 + 단순 기울기 계산 |

**분석 구조:**

```
    조절변수 (W)
        │
        ▼
    X ──────→ Y
    
    Y = b0 + b1*X + b2*W + b3*(X×W) + e
    
    상호작용항 b3가 유의하면 → 조절효과 있음
```
