# 프론트엔드 명세

> 본 문서는 React (Vite) 기반 프론트엔드의 화면 구성, 주요 컴포넌트, UI/UX 요구사항을 정의한다.  
> **UX 패턴은 Spotfire 웹 버전을 참고**하여, 단일 워크스페이스 + Flyout 패널 + Data Canvas 방식을 채택한다.

---

## 1. UX 설계 원칙 (Spotfire 참고)

### 1.1 핵심 패턴

| 패턴 | Spotfire 원칙 | 우리 프로젝트 적용 |
|---|---|---|
| **단일 워크스페이스** | 페이지 이동 없이 하나의 화면에서 모든 작업 | 프로젝트 진입 후 단일 대시보드, 단계별 뷰 전환 |
| **Flyout 패널** | 사이드에서 슬라이드 인/아웃, 메인 캔버스 유지 | 설정/기준치/결과를 Flyout에서 처리 |
| **Data Canvas** | 노드 기반 데이터 리니지 시각화 | 분석 파이프라인을 노드 그래프로 시각화 |
| **Progressive Disclosure** | 기본 옵션만 보이고, 고급 옵션은 펼침 | 기준치 "기본/고급" 토글, 결과 "요약/상세" 토글 |
| **AI 추천 (💡)** | 데이터 기반 자동 추천 | PROCESS Model 자동 판별, 기준 미달 항목 하이라이트 |

## 2. 화면 구조

### 2.1 화면 목록

> 페이지 이동이 아닌 **단일 워크스페이스 내 뷰 전환** 방식

| 번호 | 뷰 | 경로 | 설명 |
|:---:|---|---|---|
| 0 | 로그인 | `/login` | ID/PW 로그인 (유일한 별도 페이지) |
| 1 | 프로젝트 홈 | `/` | 프로젝트 목록, 최근 활동 |
| 2 | 워크스페이스 | `/projects/:id` | **단일 페이지** — 아래 모든 단계를 포함 |

### 2.2 워크스페이스 내 단계 (뷰 전환)

| 단계 | 뷰 ID | 좌측 네비 | 중앙 캔버스 | 우측 Flyout |
|:---:|---|---|---|---|
| 1 | `upload` | 파일 목록 | 데이터 미리보기 테이블 | 파일 정보, 컬럼 통계 |
| 2 | `cleansing` | 클린징 규칙 목록 | 데이터 전/후 비교 | 클린징 설정 패널 |
| 3 | `mapping` | 엑셀 컬럼 목록 | Drag & Drop 매핑 보드 | 변수 그룹 상세 |
| 4 | `model` | 변수 그룹 패널 | React Flow 모형 캔버스 | PROCESS Model 판별 결과 |
| 5 | `factor` | 변수 그룹 선택 | 요인 행렬 + 자동 분석 로그 | 기준치 설정, 분석 이력 |
| 6 | `analysis` | 분석 타입 탭 목록 | 분석 결과 테이블/차트 | 분석 옵션, AI 해석 |
| 7 | `export` | 보고서 설정 | 보고서 미리보기 | 출력 옵션, 다운로드 |

**Data Canvas 인터랙션:**

| 조작 | 동작 |
|---|---|
| **노드 클릭** | 해당 단계로 중앙 캔버스 전환 |
| **노드 호버** | 해당 단계의 요약 정보 툴팁 표시 |
| **노드 우클릭** | 단계별 컨텍스트 메뉴 (재실행, 결과 보기) |
| **줌 인/아웃** | 파이프라인 전체/부분 보기 |
| **접기/펼치기** | 하단 패널 높이 조절 (최소화 시 1줄 진행률 바) |

### 3.9 분석기법별 Flyout 패널 옵션

각 분석 탭을 선택할 때마다 우측 Flyout 패널의 **[분석 옵션]**이 아래와 같이 동적으로 변경됩니다.

| 분석 기법 | Flyout 주요 옵션 항목 | 설명 / 조작 |
|---|---|---|
| **빈도분석** | - 대상 변수 선택<br>- 차트 종류 | - 성별, 연령 등 인구통계 변수 다중 선택<br>- 막대그래프, 파이차트 등 표시 여부 |
| **신뢰도 분석** | - 대상 요인 선택<br>- 항목 제거 시 α값 표시 여부 | - 요인분석에서 확정된 요인들 중 선택<br>- Cronbach's α를 높이기 위한 항목 제거 시뮬레이션 표시 |
| **상관관계** | - 상관계수 종류<br>- 유의수준 표시 방식 | - Pearson (연속형) / Spearman (서열형) 선택<br>- p < .05 (*), p < .01 (**) 별표 표시 토글 |
| **독립표본 T검정** | - 그룹 변수 (독립변수)<br>- 검정 변수 (종속변수)<br>- 등분산 가정 | - 그룹을 나눌 범주형 변수 (예: 성별) 선택<br>- 비교할 연속형 변수 선택<br>- Levene의 등분산 검정에 따른 Welch's T-test 자동 적용 토글 |
| **일원배치 분산분석(ANOVA)** | - 그룹 변수 (3집단 이상)<br>- 검정 변수<br>- 사후검정(Post-hoc) 방법 | - 범주형 변수 (예: 연령대, 직급) 선택<br>- Scheffe, Tukey, Duncan 등 사후검정 방법 선택 |
| **다중 회귀분석** | - 종속변수 (Y)<br>- 독립변수 (X) 목록<br>- 분석 방법 (입력/단계선택) | - Y로 사용할 요인 선택<br>- X로 사용할 요인들 다중 선택<br>- Stepwise(단계선택법) 또는 Enter(동시입력) 방식 선택<br>- 다중공선성(VIF) 검사 토글 |
| **매개효과 분석** | - X, Y, M 변수 선택<br>- PROCESS Macro Model 번호<br>- 부트스트래핑 샘플 수 | - 연구모형 빌더에서 설정한 변수가 자동 세팅됨<br>- Model 4 (기본 매개) 자동 선택<br>- Bootstrapping 수 설정 (기본 5,000회) |
| **조절효과 분석** | - X, Y, W 변수 선택<br>- PROCESS Macro Model 번호<br>- 평균중심화 (Mean Centering) | - Model 1 (기본 조절), Model 7 (조절된 매개) 등 자동 선택<br>- 다중공선성 방지를 위한 평균중심화 적용 토글<br>- 조절효과 그래프(Simple Slope) 출력 설정 |

**변수 색상 규칙:**

| 변수 유형 | 색상 | 아이콘 |
|---|---|---|
| 독립변수 (IV) | 🔵 파랑 | ■ |
| 종속변수 (DV) | 🔴 빨강 | ● |
| 매개변수 (MED) | 🟢 초록 | ◆ |
| 조절변수 (MOD) | 🟠 주황 | ▲ |

## 6. WebSocket 클라이언트 설계

```typescript
// services/websocket.ts
class AnalysisWebSocket {
  private ws: WebSocket | null = null;
  
  connect(jobId: string, token: string) {
    this.ws = new WebSocket(
      `ws://localhost:8000/ws/analysis/${jobId}?token=${token}`
    );
    
    this.ws.onmessage = (event) => {
      const data = JSON.parse(event.data);
      switch (data.type) {
        case 'progress':
          // 진행률 업데이트
          break;
        case 'result':
          // 분석 결과 수신
          break;
        case 'error':
          // 에러 처리
          break;
      }
    };
  }
  
  send(action: string, payload: any) {
    this.ws?.send(JSON.stringify({ action, ...payload }));
  }
  
  disconnect() {
    this.ws?.close();
  }
}
```
