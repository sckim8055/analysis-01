## 1. 개발 단계 (5 Phase)

```
Phase 1: 기반 구축           (약 2주)
Phase 2: 데이터 처리          (약 2주)
```

## 4. PDCA 사이클 적용

### 4.1 PDCA 주기

| 주기 | 적용 단위 |
|---|---|
| **일간** | 코딩 → 테스트 → 리뷰 → 개선 (Atom 원칙 = 작은 단위) |
| **주간** | 주차 목표 달성 여부 점검 → 다음 주 계획 조정 |
| **Phase 별** | 마일스톤 달성 여부 → 다음 Phase 계획 수정 |

### 4.2 PDCA 체크리스트 (주간)

```markdown
## 주간 PDCA — Week N

### Plan (이번 주 계획)
- [ ] 구현할 기능 목록
- [ ] 작성할 테스트 목록
- [ ] 예상 산출물

### Do (실행)
- [ ] 구현 내용
- [ ] 작성한 테스트
- [ ] 커밋 이력

### Check (점검)
- [ ] 테스트 통과 여부
- [ ] SPSS 비교 결과
- [ ] 코드 품질 (Clean Code 준수 여부)
- [ ] 문서 업데이트 여부

### Act (개선)
- [ ] 발견된 문제
- [ ] 개선 조치
- [ ] 다음 주 계획 반영 사항
```

## 5. Atom 원칙 적용

일론 머스크의 Atom 원칙: **"모든 것을 최소 단위로 분해하고, 불필요한 것은 제거한다."**

| 적용 | 방법 |
|---|---|
| **기능 분해** | 각 분석을 독립 모듈로 분해 (요인분석 → KMO 계산, 요인추출, 회전, 적재값 필터링) |
| **테스트 분해** | 하나의 테스트는 하나만 검증 |
| **커밋 분해** | 하나의 커밋은 하나의 변경만 포함 |
| **불필요 제거** | 사용하지 않는 코드/기능은 즉시 제거 |
| **진행 검증** | 각 Atom 단위 완료 시 즉시 테스트 |

## 6. 리스크 관리

| 리스크 | 영향 | 확률 | 대응 |
|---|---|---|---|
| SPSS 결과와 불일치 | 높음 | 중간 | 라이브러리별 알고리즘 차이 조사, 오차 허용 범위 정의 |
| PROCESS Macro 재현 어려움 | 높음 | 중간 | Bootstrap seed 고정, 공식 논문 참고 |
| React Flow 커스텀 난이도 | 중간 | 낮음 | 대안: Cytoscape.js, D3.js |
| 대용량 데이터 성능 | 중간 | 낮음 | Celery 비동기 처리 + 결과 캐싱 |
| AI API 비용/할당량 | 낮음 | 중간 | 캐싱 + 요청 제한 + 사용자별 할당 |
