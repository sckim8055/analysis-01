# AI 사용 정책

> 본 문서는 박사논문 분석 솔루션에서 AI(Gemini, Claude API)를 활용하는 범위, 제한, 표절 대응 정책을 정의한다.

---

## 1. 목적

AI는 **분석 결과의 해석 보조** 용도로만 사용한다.
통계 수치 자체를 AI가 생성하거나 변경하는 것은 허용하지 않는다.

---

## 2. AI 활용 범위

### 2.1 허용

| 용도 | 설명 |
|---|---|
| 분석 결과 해석 초안 | 통계 분석 결과를 학술적 문장으로 변환하는 초안 생성 |
| 결과 비교 설명 | 가설 채택/기각에 대한 설명 초안 |
| 표 해석 보조 | 상관계수 행렬, 회귀분석 표 등의 의미 설명 |
| 학술 용어 제안 | 적절한 학술적 표현 추천 |

### 2.2 금지

| 금지 항목 | 사유 |
|---|---|
| 통계 수치 생성 | AI 환각(hallucination)으로 잘못된 수치 생성 위험 |
| 연구 가설 변조 | 결과에 맞춰 가설을 변경하는 것은 연구 윤리 위반 |
| 분석 방법 자동 선택 | 연구자가 직접 판단해야 하는 영역 |
| 해석 초안 무수정 사용 | 카피킬러 통과 불가, 연구자 고유성 부재 |

---

## 3. AI 호출 프로세스

```
[1] 분석 완료 (매개/조절/회귀 등)
    ↓
[2] 사용자가 "AI 해석" 버튼 클릭
    ↓
[3] 시스템이 실제 분석 결과(수치, 표)를 프롬프트에 포함
    ↓
[4] Gemini 또는 Claude API 호출
    ↓
[5] AI 해석 초안 생성 → 화면에 표시
    ↓
[6] 사용자가 초안을 검토/수정/편집
    ↓
[7] 수정본을 "확정"하여 보고서에 포함
```

---

## 4. 프롬프트 정책

### 4.1 프롬프트 구조

```
[역할]
당신은 사회과학 연구 방법론 전문가입니다.
아래 통계 분석 결과를 학술 논문 형식으로 해석해주세요.

[규칙]
- 제공된 수치만 사용하십시오. 새로운 수치를 만들지 마십시오.
- 학술 논문 결과 섹션에 적합한 문체를 사용하십시오.
- 가설 채택/기각 판단을 포함하십시오.
- p-value 기준: * p < 0.05, ** p < 0.01, *** p < 0.001

[분석 결과 데이터]
{실제 분석 결과 JSON}

[가설]
{사용자가 입력한 연구 가설}
```

### 4.2 프롬프트 안전장치

| 장치 | 설명 |
|---|---|
| 수치 검증 | AI 응답에 포함된 수치가 원본 분석 결과와 일치하는지 자동 검증 |
| 환각 감지 | 원본에 없는 수치가 발견되면 경고 표시 |
| 프롬프트 주입 방지 | 사용자 입력(가설 등)에 대한 sanitization |
| API 키 보호 | 서버 사이드에서만 API 호출, 클라이언트에 키 노출 금지 |

## 5. AI 제공자 설정

| 항목 | Gemini | Claude |
|---|---|---|
| API | Google Generative AI | Anthropic API |
| 모델 | gemini-2.5-flash 이상 | claude-sonnet-4 이상 |
| 용도 | 기본 해석 | 심층 해석 (필요 시) |
| 비용 | 저비용 | 중비용 |
| 선택 | 사용자가 프로젝트 설정에서 선택 | |

## 6. 표절 대응 정책 (카피킬러 통과 전략)

### 6.1 원칙

> 논문 제출 시 카피킬러 검색 통과가 중요하므로, AI 생성 문장을 **그대로 사용하지 않는다.**

| 원칙 | 설명 |
|---|---|
| AI 초안 ≠ 최종 문장 | AI 생성 결과는 참고용 초안일 뿐, 최종 문장은 연구자가 직접 작성 |
| 반드시 수정 필수 | 초안을 그대로 "확정"하려 하면 경고 메시지 표시 |
| 초안 vs 최종 구분 | AI 초안과 사용자 수정본을 각각 별도 저장하여 추적 가능 |
| 수정률 표시 | AI 초안 대비 사용자 수정률(%) 표시 |

### 6.2 구현 방법

```
[AI 초안]
"매개효과 분석 결과, 서비스 품질은 신뢰를 매개로 고객 만족에
유의한 간접효과를 미치는 것으로 나타났다."

     ↓ 사용자 수정 ↓

[사용자 수정본]
"본 연구의 분석 결과에 따르면, 서비스 품질이 고객 만족도에 미치는
영향 관계에서 신뢰가 유의미한 매개역할을 수행하는 것으로 확인되었다."

수정률: 68% → ✅ 충분히 수정됨
```

### 6.3 경고 시스템

| 수정률 | 상태 | 메시지 |
|---|---|---|
| 0% (무수정) | 🔴 위험 | "AI 초안을 그대로 사용하면 표절 위험이 있습니다. 수정해주세요." |
| 1~30% | 🟡 주의 | "수정이 부족합니다. 더 많은 수정을 권장합니다." |
| 31~50% | 🟢 양호 | "적절한 수준의 수정이 이루어졌습니다." |
| 51%+ | ✅ 우수 | "충분히 수정되었습니다." |

## 7. 데이터 보호

| 항목 | 정책 |
|---|---|
| AI에 전송되는 데이터 | 분석 결과 수치만 전송, 원본 설문 데이터는 전송하지 않음 |
| 개인정보 | AI 프롬프트에 개인식별정보(PII) 포함 금지 |
| API 로깅 | AI 요청/응답을 서버에 로깅하여 감사 추적 가능 |
| 데이터 보존 | AI 초안과 수정본 모두 DB에 저장, 프로젝트 삭제 시 함께 삭제 |

---

## 8. 비용 관리

| 항목 | 정책 |
|---|---|
| 사용자당 일일 제한 | 최대 20회 API 호출 |
| 응답 캐싱 | 동일 분석 결과에 대한 반복 요청은 캐시 응답 |
| 모델 선택 | 기본 Gemini (저비용), 필요 시 Claude (심층) |
| 비용 모니터링 | 월별 API 비용 대시보드 (관리자용) |
