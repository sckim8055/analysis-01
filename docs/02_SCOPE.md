# 프로젝트 범위 정의

> 본 문서는 프로젝트의 범위(In-Scope / Out-of-Scope)와 제약사항 및 가정사항을 정의한다.

---

## 1. In-Scope (프로젝트 범위 내)

### 1.1 데이터 처리
- Excel(.xlsx, .xls), CSV 파일 업로드 및 서버 저장
- 데이터 미리보기 (상위 N행)
- 데이터 클린징 (결측치 처리, 이상치 탐지, 역코딩)
- 리커트 척도 기반 설문 데이터 처리

### 1.2 변수 관리
- 변수 유형 분류: 독립변수, 종속변수, 매개변수, 조절변수
- 각 변수의 하위변수(측정 항목) 매핑
- 변수 그룹 저장/수정/삭제

### 1.3 연구모형 설계
- Drag & Drop 기반 연구모형 빌더
- 노드(변수) 간 관계(선) 연결
- PROCESS Macro v5 Model 자동 판별 (Model 1, 4, 7, 8, 14 등)
- 연구모형 저장/불러오기

### 1.4 통계 분석 (SPSS 26 기준 대체)
- 탐색적 요인분석 (EFA) / 주성분분석 (PCA)
- 신뢰도 분석 (Cronbach's α)
- 상관관계 분석 (Pearson, Spearman)
- 독립표본 T검정
- ANOVA (일원분산분석)
- 다중회귀분석 (OLS)
- 매개효과 분석 (Bootstrap 기반)
- 조절효과 분석 (상호작용항 + 단순 기울기)

### 1.5 AI 해석 보조
- 분석 결과를 기반으로 Gemini / Claude API 호출
- 결과 해석 초안 생성
- 사용자 편집 기능
- 카피킬러 통과 고려 가이드라인

### 1.6 보고서 출력
- Word(.docx) 출력 — 논문 양식에 적합한 표/텍스트
- Excel(.xlsx) 출력 — 분석 결과 데이터 표

### 1.7 인증
- ID/PW 기반 로그인 (JWT)
- 관리자가 백엔드에서 계정 생성

## 2. Out-of-Scope (프로젝트 범위 외)

| 항목 | 제외 사유 |
|---|---|
| 회원가입 UI | 소수 사용자, 관리자가 계정 생성 |
| 비밀번호 찾기/재설정 UI | 관리자에게 직접 요청 |
| 구조방정식 (SEM) | 현재 범위 외, 추후 확장 가능 |
| 확인적 요인분석 (CFA) | 현재 범위 외, 추후 확장 가능 |
| 다수준 분석 (HLM) | 현재 범위 외 |
| 실시간 협업 (다중 사용자 동시 편집) | 소수 사용자이므로 불필요 |
| 모바일 최적화 | 데스크톱 웹 브라우저 대상 |
| PDF 출력 | Word/Excel 우선, 추후 확장 가능 |
| 다국어 지원 | 한국어 단일 |

---

## 3. 제약사항

| 구분 | 내용 |
|---|---|
| 분석 기준 | SPSS 26 버전 출력 결과와 비교 검증 필요 |
| 매개/조절 기준 | PROCESS Macro v5 결과와 비교 검증 필요 |
| 논문 제출 | 카피킬러 통과를 위해 AI 생성 문장 직접 사용 금지 |
| 사용자 수 | 소수 (10명 이하), 대규모 동시 접속 고려 불필요 |
| 데이터 규모 | 설문 데이터 기준 수백~수천 행 수준 |
| 보안 | 개인정보 포함 가능성 있으므로 서버 접근 통제 필요 |

---

## 4. 가정사항

| 구분 | 내용 |
|---|---|
| 데이터 형식 | 리커트 척도(5점/7점) 기반 설문 데이터를 주로 다룸 |
| 운영 환경 | Linux 서버 또는 Docker 기반 배포 |
| 브라우저 | Chrome, Edge 등 최신 브라우저 대상 |
| Python | 3.11 이상 |
| Node.js | 18 이상 |
| 분석 결과 신뢰도 | SPSS 26과의 결과 비교로 검증 완료 후 사용 |

---

## 5. 향후 확장 가능 영역

> 현재는 범위 외이나, 아키텍처 설계 시 확장 가능성을 고려하는 항목

- 확인적 요인분석 (CFA) — `semopy` 라이브러리 활용 가능
- 구조방정식 (SEM) — `semopy` 또는 `lavaan` R 연동
- 경로분석 — `statsmodels` 확장
- 다수준 분석 (HLM) — `statsmodels` MixedLM
- PDF 보고서 출력 — `reportlab` 또는 `weasyprint`
- 영문 논문 지원 — i18n 적용
